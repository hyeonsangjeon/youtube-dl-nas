# youtube-dl-nas v2 Renewal Project

## 프로젝트 개요
개인 미디어 아카이브 플랫폼. 단순 다운로더 → 구독·자동수집·정리·검색·스트리밍으로 진화 중.
- 레포: https://github.com/hyeonsangjeon/youtube-dl-nas
- 현재 v1이 master에서 운영 중 (Docker Hub에서 사용자들이 pull하여 사용)

## 기술 스택 (v2)
- Backend: **FastAPI** + **SQLAlchemy (async)** + **SQLite (aiosqlite)** + **Alembic**
- Frontend: React 18 + Tailwind CSS + Vite (Phase 1 backend 완료 후 착수)
- Auth: JWT (access + refresh token)
- WebSocket: FastAPI native async
- Container: Docker multi-stage build

## 브랜치 전략
```
master (프로덕션 — 직접 push 금지)
  └── develop (통합 테스트)
        ├── feat/phase1-backend
        ├── feat/phase1-frontend
        └── feat/phase1-ci
```

## 디렉토리 경계 (충돌 방지)
- `backend/` — 백엔드 에이전트 전용
- `frontend/` — 프론트엔드 에이전트 전용
- `.github/`, `Dockerfile`, `docker-compose.*` — CI/인프라 에이전트 전용
- 루트의 v1 파일들 (`youtube-dl-server.py`, `Auth.json` 등) — **절대 수정 금지**

## 커밋 컨벤션
Conventional Commits:
```
feat(backend): 설명
fix(backend): 설명
chore(ci): 설명
test(backend): 설명
docs: 설명
```
스코프: `backend`, `frontend`, `ci`, `docker`, `docs`

## ⚠️ 절대 금지 사항
1. v1 코드 수정 금지 (youtube-dl-server.py, Auth.json, run.sh, Dockerfile, requirements.txt, static/, bottle_websocket/)
2. master 브랜치 직접 push 금지
3. backend/ 밖에 파일 생성 금지 (백엔드 작업 시)
4. frontend/ 밖에 파일 생성 금지 (프론트엔드 작업 시)

## 레거시 API 하위호환 (필독)
외부 개발자들이 사용 중인 REST API를 반드시 유지해야 한다:
```
POST /youtube-dl/rest
Body: {"url": "...", "resolution": "best", "id": "myid", "pw": "mypw"}
Response: {"success": true, "msg": "download has started", "Remaining downloading count": "0"}
```
v2에서 `routers/legacy.py`로 동일 스펙 유지.
