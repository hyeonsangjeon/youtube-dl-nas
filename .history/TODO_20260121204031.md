5) Azure OpenAI Responses API 기반 “Subtitle Summary Agent” 도입

목표: SRT 다운로드 → 정제 → 타입별 요약 → md 생성/다운로드를 **tool calling(함수 호출)**로 오케스트레이션.
(모놀리식/단일 워커 설계 유지, 과한 멀티에이전트/장기 루프 금지)

5.1 Responses API 호출 형태(요청/루프)
	•	Azure OpenAI Responses API 사용 (base_url: https://{resource}.openai.azure.com/openai/v1/)  ￼
	•	1회 요청에서 모델이 tool 호출을 제안하면:
	•	서버가 해당 tool 실행 → tool 결과를 다시 모델에 입력 → 다음 액션 결정
	•	루프 제한(가드레일):
	•	최대 iteration: MAX_AGENT_ITERS = 3
	•	최대 tool 호출 수: MAX_TOOL_CALLS = 4 (download/clean/summarize/render_md)
	•	최대 입력 SRT 길이 제한(토큰/문자 기준): 예) 200k chars 초과 시 chunking 또는 요약 전 축약 필수
	•	전체 처리 타임아웃: 예) 60~120초 (SRT 다운로드 포함)
	•	실패 시 “부분 결과라도 반환” + 사용자에게 이유/대안 제공

5.2 Tool JSON Schema 정의(Responses API tools[])
Responses API의 tools에 아래 4개 function tool을 등록. (JSON Schema 기반)  ￼

(1) download_subtitle

{
  "type": "function",
  "function": {
    "name": "download_subtitle",
    "description": "Download subtitles (SRT preferred) for a given YouTube URL and language code. Returns local file path and raw subtitle text.",
    "parameters": {
      "type": "object",
      "properties": {
        "url": { "type": "string", "description": "YouTube video URL" },
        "lang": { "type": "string", "description": "Subtitle language code (e.g., ko, en, ja). Use user input." },
        "prefer_auto_caption": { "type": "boolean", "description": "If true, allow auto-generated captions when official subtitles are not available.", "default": false }
      },
      "required": ["url", "lang"]
    }
  }
}

(2) clean_subtitle

{
  "type": "function",

5.3 System Prompt (에이전트 지시문)
Responses API 요청의 시스템 메시지(또는 developer/system role)에 아래 핵심을 포함:
	•	너는 “Subtitle Summary Agent”다.
	•	사용자가 제공한 URL/언어/요약 타입으로 작업한다.
	•	반드시 다음 순서로 tool을 사용하라:
	1.	download_subtitle
	2.	clean_subtitle
	3.	summarize
	4.	render_markdown
	•	자막이 없으면:
	•	prefer_auto_caption을 true로 재시도(단 1회)
	•	그래도 실패하면 사용자에게 “자막 없음” 안내 + 대안 제시(다른 언어/자동자막)
	•	요약은 사실 기반, 자막에 없는 내용은 추측하지 말 것.
	•	최종 응답에는:
	•	화면에 보여줄 요약(타입별 섹션)
	•	md 다운로드 링크(서버가 제공하는 파일 경로/엔드포인트로 매핑)
	•	비용/지연 최소화를 위해:
	•	불필요한 재시도 금지
	•	iteration/툴콜 제한 준수
	•	긴 자막은 먼저 압축/정제 후 요약

(참고: tool/function calling은 JSON schema 기반으로 정의한 tool을 모델이 선택 호출하도록 하는 패턴)  ￼

5.4 루프/비용 가드레일(필수)
	•	루프 제한
	•	MAX_AGENT_ITERS = 3
	•	MAX_TOOL_CALLS = 4
	•	토큰/비용 제한(권장)
	•	요청당 max_output_tokens 상한 설정
	•	자막 길이 기준으로 summarize 입력 텍스트 상한(예: 80k~120k chars)
	•	초과 시: “압축 요약(1차) → 타입별 요약(2차)” 2단계로 하되, iteration 3을 넘지 않음
	•	타임아웃/실패 처리
	•	다운로드 실패/LLM 실패 시: “부분 결과(가능하면)” + 에러 메시지
	•	key/endpoint 미설정이면: 설정 화면으로 리다이렉트
	•	보안
	•	key는 로그/에러 메시지에 절대 노출 금지
	•	tool 결과에 key 포함 금지

5.5 UI 연동(진행상태 표시)
	•	WebSocket(있다면)으로 단계별 상태 push:
	•	DOWNLOADING_SUBTITLE → CLEANING → SUMMARIZING → RENDERING_MD → DONE/FAILED
	•	최소 구현: 폴링/로딩 스피너 + 단계 텍스트

5.6 Responses API 실제 request payload 예시 (Python/Bottle 우선)

(A) Python(OpenAI SDK) 최소 예시: tools + system prompt + 1턴 호출

import os
from openai import OpenAI

AZURE_OPENAI_ENDPOINT = os.getenv("AZURE_OPENAI_ENDPOINT")  # e.g. https://YOUR-RESOURCE-NAME.openai.azure.com
AZURE_OPENAI_API_KEY  = os.getenv("AZURE_OPENAI_API_KEY")
MODEL_DEPLOYMENT      = os.getenv("AZURE_OPENAI_DEPLOYMENT")  # e.g. o4-mini (deployment name)

client = OpenAI(
    base_url=f"{AZURE_OPENAI_ENDPOINT}/openai/v1/",
    api_key=AZURE_OPENAI_API_KEY,
)

SYSTEM_PROMPT = """You are Subtitle Summary Agent...
(see section 5.3)"""

TOOLS = [
  # (see 5.2 JSON schema 4 tools)
]

def call_agent_once(url: str, lang: str, summary_types: list[str]):
    user_input = {
        "type": "input_text",
        "text": f"URL={url}\nLANG={lang}\nSUMMARY_TYPES={summary_types}"
    }

    resp = client.responses.create(
        model=MODEL_DEPLOYMENT,
        input=[
            {"role": "system", "content": SYSTEM_PROMPT},
            {"role": "user", "content": [user_input]},
        ],
        tools=TOOLS,
        # 가드레일
        max_output_tokens=1200,
    )
    return resp

	•	참고: Azure OpenAI에서도 OpenAI(base_url="https://...openai.azure.com/openai/v1/", api_key=...) + client.responses.create(...) 패턴을 사용  ￼

(B) Python(Bottle)에서 “툴 호출 루프” 처리 요약 (MAX 3 iters)
아래는 “구현 요약” 형태로 TODO에 남기고, 실제 코드는 youtube-dl-server.py에 구현

	•	루프 구조:
	•	resp = responses.create(...)
	•	resp 안에 “tool call 요청”이 있으면:
	1.	download_subtitle/clean_subtitle/summarize/render_markdown 중 하나 실행
	2.	tool 결과를 “tool output”으로 다시 responses.create(...)에 입력
	•	최대 3회 반복 후 종료

MAX_AGENT_ITERS = 3

def run_agent(url, lang, summary_types):
    # 1) 최초 호출
    resp = call_agent_once(url, lang, summary_types)

    for _ in range(MAX_AGENT_ITERS):
        tool_calls = extract_tool_calls(resp)  # TODO: SDK 응답 구조에 맞춰 파싱
        if not tool_calls:
            return resp  # 최종 자연어 결과

        tool_outputs = []
        for tc in tool_calls:
            # tc.name, tc.arguments 파싱
            result = dispatch_tool(tc)  # 실제 yt-dlp/SRT 처리 + 요약 + md 생성
            tool_outputs.append({
                "type": "tool_output",
                "tool_call_id": tc.id,
                "output": result,  # JSON-serializable
            })

        # 2) tool 결과를 넣어서 다음 턴 진행
        resp = client.responses.create(
            model=MODEL_DEPLOYMENT,
            input=[
                {"role": "system", "content": SYSTEM_PROMPT},
                {"role": "user", "content": [{"type":"input_text","text": f"URL={url}\nLANG={lang}\nSUMMARY_TYPES={summary_types}"}]},
                {"role": "tool", "content": tool_outputs},  # TODO: 실제 SDK 포맷 확인 후 반영
            ],
            tools=TOOLS,
            max_output_tokens=1200,
        )

    # 루프 초과 시: 부분 결과 + 실패 메시지
    return resp

	•	구현 시 주의:
	•	“툴콜 응답 구조(extract_tool_calls)”와 “tool output 입력 포맷”은 현재 openai SDK 응답 스키마에 맞춰 확정 필요 (docs 기준으로 responses는 function calling 지원)  ￼
	•	로그에 key/비밀 값 절대 출력 금지

5.7 Responses API 실제 request payload 예시 (JS/REST: fetch 또는 curl)

(A) Node/JS fetch 예시 (서버 사이드)

const endpoint = process.env.AZURE_OPENAI_ENDPOINT; // https://YOUR-RESOURCE-NAME.openai.azure.com
const apiKey = process.env.AZURE_OPENAI_API_KEY;
const model = process.env.AZURE_OPENAI_DEPLOYMENT;

const body = {
  model,
  input: [
    { role: "system", content: "You are Subtitle Summary Agent...(see 5.3)" },
    { role: "user", content: [{ type: "input_text", text: "URL=...\nLANG=ko\nSUMMARY_TYPES=[\"short\",\"action_items\"]" }] }
  ],
  tools: [ /* 5.2의 4개 tool schema */ ],
  max_output_tokens: 1200
};

const res = await fetch(`${endpoint}/openai/v1/responses`, {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "Authorization": `Bearer ${apiKey}`
  },
  body: JSON.stringify(body)
});

const json = await res.json();
console.log(json);
